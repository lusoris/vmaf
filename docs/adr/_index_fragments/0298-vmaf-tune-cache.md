| [ADR-0298](0298-vmaf-tune-cache.md) | vmaf-tune content-addressed encode/score cache (parent: [ADR-0237](0237-quality-aware-encode-automation.md)). Adds `tools/vmaf-tune/src/vmaftune/cache.py` keyed on `sha256(src_content) + encoder + preset + crf + adapter_version + ffmpeg_version`. Cache lives at `$XDG_CACHE_HOME/vmaf-tune/` (or `~/.cache/vmaf-tune/`) with LRU eviction at a 10 GiB default cap. Wired into the corpus loop with default-ON; `--no-cache` / `--cache-dir` / `--cache-size-gb` opt-out and override. JSONL row schema unchanged — cache is an opaque sidecar, not baked into the row. Re-runs of the same `(preset, crf)` sweep collapse from minutes (full re-encode + re-score) to milliseconds (filesystem reads). | Accepted | tools, vmaf-tune, cache, fork-local |
