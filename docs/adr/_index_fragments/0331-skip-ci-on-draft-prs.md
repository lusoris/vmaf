| [ADR-0331](0331-skip-ci-on-draft-prs.md) | Skip CI on draft pull requests across all 8 fork workflows. Adds `types: [opened, synchronize, reopened, ready_for_review]` to each `pull_request` trigger and gates every top-level job with an `if:` clause that skips when `github.event.pull_request.draft == true` while keeping `push:` triggers intact. The `ready_for_review` event re-fires CI when a draft is promoted; draft PRs are unmergeable by GitHub's definition, so the resulting "Required check missing" state on drafts is benign. Cuts CI spend by roughly half on the fork's typical 10+-draft work-in-progress queue. | Proposed | ci, build, fork-local |
