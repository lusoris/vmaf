| [ADR-0299](0299-vmaf-tune-gpu-score.md) | Add `--score-backend {auto\|cpu\|cuda\|sycl\|vulkan}` to `vmaf-tune corpus` (default `auto`). Wires the libvmaf CLI's existing `--backend NAME` selector into the Phase A scoring step, removing the CPU-only score floor that capped corpus throughput at 1–2 fps on 1080p sources. New module `tools/vmaf-tune/src/vmaftune/score_backend.py` parses `vmaf --help` for advertised backends, intersects with cheap host probes (`nvidia-smi -L`, `vulkaninfo --summary`, `sycl-ls`), and walks the fallback chain `cuda → vulkan → sycl → cpu` in `auto` mode. Explicit non-auto requests are strict — a `--score-backend cuda` on a CPU-only host raises `BackendUnavailableError` with a diagnostic; we never silently downgrade. Ships docs/usage update + 22-test unit coverage that mocks help-text and probe runners.
