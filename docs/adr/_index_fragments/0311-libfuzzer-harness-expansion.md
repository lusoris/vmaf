| [ADR-0311](0311-libfuzzer-harness-expansion.md) | libFuzzer harness expansion — `fuzz_yuv_input` + `fuzz_cli_parse` (extends [ADR-0270](0270-fuzzing-scaffold.md)). Adds two new harnesses to `libvmaf/test/fuzz/`: [`fuzz_yuv_input.c`](../../libvmaf/test/fuzz/fuzz_yuv_input.c) wraps the headerless raw-YUV reader (`tools/yuv_input.c`) via `raw_input_open` against a fixed 32×32 envelope rotating through six (pix_fmt, bitdepth) combos, up to 8 frames per iteration; [`fuzz_cli_parse.c`](../../libvmaf/test/fuzz/fuzz_cli_parse.c) wraps the CLI argv tokeniser + colon-delimited `parse_model_config` / `parse_feature_config` sub-parsers (`tools/cli_parse.c`) via a NUL-separated argv vector (capped at 64 args) and a `-Wl,--wrap=exit` linker shim with `setjmp`/`longjmp` trampoline so `usage()`'s `exit(1)` does not terminate the fuzzer process. Each harness ships 6 hand-crafted seeds; nightly [`fuzz.yml`](../../.github/workflows/fuzz.yml) matrix gains both targets at 60 s/harness/night. Operator runbook updated in [`docs/development/fuzzing.md`](../development/fuzzing.md).
