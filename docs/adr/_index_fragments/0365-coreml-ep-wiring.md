| [ADR-0365](0365-coreml-ep-wiring.md) | Wire the Apple CoreML execution provider into the tiny-AI ORT dispatch layer. Adds four `--tiny-device` selectors (`coreml`, `coreml-ane`, `coreml-gpu`, `coreml-cpu`) and the matching `VmafDnnDevice` enum values 5..8 (append-only). The `coreml-ane` selector pins `MLComputeUnits=CPUAndNeuralEngine` for highest perf-per-watt on M-series silicon; the unscoped `coreml` lets the EP auto-route. Wiring uses the generic key/value `SessionOptionsAppendExecutionProvider` form so the Linux build degrades cleanly when the EP is absent. End-to-end ANE silicon validation deferred until Apple-silicon hardware access. Apple-side parallel to ADR-0332's OpenVINO NPU wiring. | Proposed | ai, dnn, coreml, apple-silicon, fork-local |
