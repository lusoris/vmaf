| [ADR-0293](0293-vmaf-tune-saliency-aware.md) | Adds saliency-aware ROI tuning to `vmaf-tune` (Bucket #2 of the PR #354 audit). New `tools/vmaf-tune/src/vmaftune/saliency.py` consumes the fork-trained `saliency_student_v1` ONNX (ADR-0286), produces a per-MB QP-offset map clamped to ±12 (matching `vmaf-roi` ADR-0247), and a new `vmaf-tune recommend --saliency-aware` subcommand wires it into the FFmpeg encode path via x264 `--qpfile`. Pure-Python ONNX inference (mocked in tests) chosen over shelling out to `vmaf-roi` so the harness ships without a built libvmaf dependency; bit-for-bit numeric contract with `vmaf-roi`'s C signal-blend pinned by test. Graceful fallback to plain encode when onnxruntime / model file missing. x264 only in this PR; x265 / SVT-AV1 are a one-file codec-adapter follow-up. Companion to ADR-0237 (`vmaf-tune` umbrella) and ADR-0286 (`saliency_student_v1`). | Accepted | tooling, ai, saliency, ffmpeg, codec, fork-local |
