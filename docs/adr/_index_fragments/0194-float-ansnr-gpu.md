| [ADR-0194](0194-float-ansnr-gpu.md) | T7-23 / batch 3 part 2 — `float_ansnr_{vulkan,cuda,sycl}` extractors. Single-dispatch GPU kernel applies the CPU's 3x3 ref filter and 5x5 dis filter inline from a 20×20 shared/SLM tile, then accumulates per-pixel `sig = ref_filtr²` and `noise = (ref_filtr - filtd)²` into per-WG float partials. Host reduces in `double` and applies the CPU formulas `float_ansnr = 10*log10(sig/noise)` and `float_anpsnr = MIN(10*log10(peak²·w·h/max(noise, 1e-10)), psnr_max)`. Edge-replicating mirror (`2*size - idx - 1`) matches CPU `ansnr_filter2d_s` — same divergence-from-motion footgun as ADR-0193. Empirical floor on cross-backend gate fixture: `max_abs_diff = 6e-6` (8-bit, 48 frames) and `2e-6` (10-bit, 3 frames) on all three backends — identical numbers across Vulkan / CUDA / SYCL. Closes ANSNR's "CPU-only float, no GPU twin" matrix gap. | Accepted | vulkan, cuda, sycl, gpu, feature-extractor, fork-local, places-4 |
