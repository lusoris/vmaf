Promotes CUDA's `libvmaf/src/cuda/ring_buffer.{c,h}` (143 LOC, callback-based round-robin pool — original Netflix code, already the right shape) to backend-agnostic `libvmaf/src/gpu_picture_pool.{c,h}`, renaming the public-internal types: `VmafRingBuffer` → `VmafGpuPicturePool`, `VmafRingBufferConfig` → `VmafGpuPicturePoolConfig`, `vmaf_ring_buffer_init`/`_close`/`_fetch_next_picture` → `vmaf_gpu_picture_pool_init`/`_close`/`_fetch`. SYCL's `vmaf_sycl_picture_pool_*` keeps its public-internal C-API but the implementation delegates to the generic pool; SYCL drops ~70 LOC of duplicated init/fetch/close + `std::mutex` plumbing (generic pool uses `pthread_mutex_t`, fine in C++ TUs). Vulkan migration bundled into this PR after [ADR-0238](0238-vulkan-picture-preallocation.md) merged — `picture_vulkan_pool.c` rewrites as a thin wrapper around the generic pool, dropping ~100 LOC. Net result: ONE round-robin implementation across CUDA / SYCL / Vulkan, plus a clean entry point for the upcoming HIP backend ([ADR-0212](0212-hip-backend-scaffold.md)) and any future Metal / DirectML port.
