## TL;DR

Audited `docs/state.md` against the last 30 days of merged PRs
(2026-04-08 — 2026-05-07, 80+ merged PRs). Per CLAUDE.md §12 r13 every
PR that closes / opens / rules-out a bug must update `docs/state.md`
in the same PR; the rule is reviewer-enforced, not CI-enforced.

**Drift summary:**

- **4 missing closure rows** — bug-fix PRs that did not touch
  `docs/state.md` and are unambiguously in scope per ADR-0165.
- **3 stale rows** — duplicate Netflix#955 entries (line 100 + 103),
  a stray `|---|---|---|---|` separator (line 102) breaking the
  Deferred-external-trigger table, and a "Deferred dataset access"
  row with `TRIGGERED 2026-04-29` content sitting in the wrong
  section (should have moved to "Recently closed" or been retired).
- **2 needs-maintainer-disposition** — items where the closure
  section / row text is ambiguous without owner context.

**Most surprising drift**: **T-VK-1.4-BUMP** is double-listed —
present in **Open bugs** (line 29, since PR #346 partially closed
Step A) **and** in **Deferred (external trigger)** (line 101). The
Deferred row was added by PR #338 (2026-05-04) but never moved /
deleted when PR #346's Step A landed and the row migrated to Open
bugs by PR #346. The two rows tell contradictory stories about the
same backlog item.

This audit + backfill is itself an in-PR exercise of CLAUDE.md
§12 r13 — the audit row gets recorded so the next session does not
have to re-derive it.

## Method

1. **PR punch list (last 30 days)**:
   ```
   gh pr list --state merged --limit 200 --search "merged:>=2026-04-08" \
     --json number,title,mergedAt,body,files --jq '...'
   ```
   Filtered for titles matching `fix|bug|crash|leak|oob|overflow|regression|drift|hang|race|deadlock|segfault|hardening|hotfix` plus
   `chore(upstream)` ports that close Netflix issues.

2. **state.md touch detection**: cross-referenced each candidate
   PR's `.files[].path` against `docs/state.md`. PRs that did not
   touch state.md and whose body did not carry a `no state delta:
   REASON` opt-out were flagged.

3. **Existing-row staleness audit**: read each row in state.md;
   verified linked PRs / commits resolve, checked for duplicate
   bug IDs across sections, looked for table-rendering damage.

4. **ADR cross-reference**: every `docs/adr/` row whose body says
   "fixes issue #X" or "closes Netflix#Y" was checked against
   state.md. Recent ADR-0309 / 0319 / 0321 chain (ensemble LOSO
   trainer + production flip) was already covered by the row PR
   #424 added at line 46.

## Assumptions, limitations, future work

- **30-day window**: only PRs merged 2026-04-08 — 2026-05-07
  audited. Older PRs are by ADR-0165 design rolling off into
  `git log`. A future audit may want to sweep the older
  "Recently closed" rows to roll off any past-90-day entries.
- **Title heuristic**: PRs without `fix|bug|...` in the title that
  nonetheless closed a bug may have escaped detection. The body
  scan partially compensates but is not exhaustive.
- **Reviewer-enforced rule**: the underlying drift exists because
  CLAUDE.md §12 r13 has no CI gate. Backlog idea: a `scripts/ci/
  state-md-touch-check.sh` that flags PRs whose title-class is
  bug-fix and that don't touch `docs/state.md` (with explicit
  `no state delta: REASON` opt-out parsed from the body).
