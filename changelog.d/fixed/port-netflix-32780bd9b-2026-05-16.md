- **Race-safe fixture download in parallel test runs.** Port of upstream
  Netflix/vmaf commit `32780bd9b` (`python/config: make download_reactively
  race-safe under parallel execution`). `download_reactively()` previously
  called `urlretrieve()` directly to `local_path`, so concurrent pytest-xdist
  workers that shared a fixture cache could observe a partially-written file
  (the second worker saw `os.path.exists()` return `True` before the first
  had finished writing and short-circuited, reading garbage). The fix
  downloads to a `tempfile.mkstemp()` sibling then calls `os.replace()`,
  which is atomic on POSIX, so readers always observe a complete file.
  Cleanup of the tempfile on error is also guaranteed via a nested
  `try/except OSError` guard.
