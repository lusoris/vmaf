Third-party license bundle for the fork-local mirror of U-2-Net's
`u2netp` checkpoint.

Per ADR-0325 (`docs/adr/0325-u2netp-fork-mirror-scaffold.md`),
the fork redistributes the upstream `xuebinqin/U-2-Net` `u2netp`
checkpoint (and/or its ONNX rewrap) as a GitHub Release asset
attached to the lusoris/vmaf repository. This file is the
Apache-2.0 §4 (a) + (c) compliance receipt — it travels with the
binary in the same release.

Upstream NOTICE file: none (verified against HEAD ac7e1c81 —
                      upstream tree carries no `NOTICE` text file,
                      so Apache-2.0 §4 (d) is moot for this
                      redistribution.)

Upstream paper citation:

  Qin, X., Zhang, Z., Huang, C., Dehghan, M., Zaiane, O.R.,
  Jagersand, M., 2020. U^2-Net: Going Deeper with Nested
  U-Structure for Salient Object Detection. Pattern Recognition.

If the redistributed asset is the upstream `.pth` checkpoint
verbatim, the weights are byte-identical to the upstream Google
Drive download; no modification beyond container-level metadata
(filename + sha256).

If the redistributed asset is an ONNX rewrap, it was produced by
`ai/scripts/export_u2netp_mirror.py` (filed as a follow-up to
ADR-0325; not present in the scaffold-only PR) running
`torch.onnx.export` at opset 17 against the upstream commit
listed above. The exporter preserves the upstream graph topology
and weight values byte-for-byte; only the container format
changes (`.pth` PyTorch state_dict → ONNX protobuf). The ONNX
graph carries a `metadata_props` block recording the upstream
commit, the export tool version, and the conversion date — so the
rewrap is auditable per Apache-2.0 §4 (b).
